- Um protocolo é um conjunto de regras que faz com que haja interação padronizada entre dois ou mais sistemas.
  
- HTTP nada mais é do que um protoloco usado na web para realizar a comunicação entre sistemas. Ele permite que cliente e o servidor interajam
de forma que haka tradução para ambas as partes.

- Protocolos padronizados são fundamentais para o funcionamento da internet pq permitem que o cliente e o servidor se comuniquem utilizando uma
linguagem padronizada.Sem isso, cada navegador precisaria de um protocolo (tradução) diferente entre o cliente e o servidor.

- Uma boa analogia poderia ser a seguinte: Cada marca de veiculo desenvolve seus carros de maneira diferente, com peças, sistemas, materiais,
e encaixe de ferramentas diferentes. Isso até soa bem para a marca (daria uma certa identidade), mas e na hora da manutenção? O mecanico teria
extrema dificuldade em aprender a trabalhar com todos os tipos de carro, ele precisaria ter um manual (um protocolo) para cada marca diferente,
alem de precisar ter ferramentas especificas de cada carro. Isso mostra o quão importante é a padronização, não só na web mas em diversos
ambiente
